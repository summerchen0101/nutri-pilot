# 飲控 + 健康商城平台 — 產品總覽

> 此文件是 Cursor 協作開發的核心知識文件。  
> 開始任何新功能前，先閱讀本文件確認方向。

---

## 產品定位

**一句話定位**：用 AI 飲食控管產生用戶黏著，再透過個人化健康商城變現。

這不是普通的卡路里計算器，也不是普通電商。  
核心差異在於：商城推薦完全基於用戶的飲食法、忌食清單、健康目標，而不是即時熱量缺口。

---

## 商業邏輯閉環

```
Onboarding 收集偏好（飲食法 + 忌食 + 目標）
        ↓
AI 每日菜單 → 用戶每天回來打卡
        ↓
拍照記錄飲食 → 降低記錄門檻
        ↓
體重趨勢讓用戶看到成效 → 信任平台
        ↓
商城根據飲食法 + 目標精準推薦商品
        ↓
訂閱補貨 → 週期性收入 + 長期黏著
```

**飛輪邏輯**：用戶資料越多 → 推薦越精準 → 轉換率越高 → 用戶越信任 → 繼續使用記錄。

## 核心功能範圍（MVP）

### 保留的功能
- Onboarding 建檔（飲食法、忌食、身體數據、目標）
- BMI / BMR / TDEE 自動計算
- AI 每日菜單生成（Claude API + Queue）
- 菜單打卡 + 換食材 AI
- 拍照辨識熱量（Claude Vision）
- 飲食記錄（搜尋輸入 + 手動）
- Dashboard（熱量圓環 + 今日餐食 + AI 今日建議）
- 體重趨勢圖 + 熱量趨勢圖
- AI 週報洞察
- 健康商城（個人化推薦排序）
- 商品詳情（符合度說明）
- 購物車 + Stripe 結帳
- 訂閱管理（Stripe Billing）

### 明確排除（不在此 codebase 內）
- 語音輸入
- 條碼掃描
- 裝置整合（Apple Health、Garmin、血糖儀）
- PDF 匯出
- 用餐 / 飲水提醒通知
- 血糖 / 血壓即時警示
- 飲水格子視覺化
- 睡眠 / 運動詳細記錄與圖表
- 成就系統
- 購物清單生成（與商城目標衝突）
- 品牌商家後台（MVP 階段用 Supabase Studio 管理）
- 用戶評價系統
- 廣告置頂加權

| 階段 | 天數 | 平台目標 |
|------|------|----------|

## 推薦引擎的核心邏輯

商城推薦不是即時熱量對比，而是長期屬性比對：

1. **飲食法符合度**（0–40 分）：商品 `dietTags` 包含用戶飲食法
2. **忌食安全**（硬過濾）：有衝突的商品直接排除，回傳 -999
3. **成分安全**（0–20 分）：忌食食材不在成分中
4. **長期目標符合**（0–15 分）：減重→低熱量商品加分，增肌→高蛋白加分
5. **回購歷史**（0–15 分）：曾購買過的加分
6. **評分**（0–10 分）：平均星等加權
