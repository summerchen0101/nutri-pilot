# 設定頁改版為我的頁與底部導覽調整

**日期**：2026-04-20  
**影響規格**：docs/09-ui-design.md  
**類型**：修改

## 原規格

- 既有 `/settings` 為傳統表單式「設定」頁，含訂閱區塊與多個輸入欄位。  
- 底部導覽為 4 個項目，設定頁顯示標籤為「設定」。

## 實際做法

- `/settings` 保留路由，頁面改為「我的」資訊卡佈局，並重整為 5 個 section：用戶資料、身體數據、飲控目標、飲食偏好、帳號管理。  
- 加入 inline 互動：名字「編輯」展開 inline 表單、身體數據「更新」展開 inline 表單。  
- 飲食方式與忌食/過敏改為 bottom sheet 編輯。  
- 底部導覽改為 5 項（總覽、記錄、商城、分析、我的），`/settings` 標籤改「我的」，並新增分析/我的 icon。  
- `MainAppShell` 新增 `SHOW_BOTTOM_NAV` 白名單，僅在 `/dashboard`、`/log`、`/shop`、`/analytics`、`/settings` 顯示 BottomNav，子頁面不顯示。

## 原因

配合最新資訊架構與視覺規範，將設定頁升級為主頁級「我的」，讓個人資訊與調整入口更集中且操作成本更低，同時對齊 5-tab 主導航結構。

## 後續

可再補上「重置數據」與「刪除帳號」的正式後端流程（目前為 UI 入口與提示）。
