# 飲食記錄「新增食物」搜尋流程 UI

**日期**：2026-04-19  
**影響規格**：docs/06-pages.md、docs/09-ui-design.md  
**類型**：修改 + 新增（資料欄位）

## 原規格

`/log` 搜尋區塊描述為 Open Food Facts 搜尋 + 結果列表 + 份量調整 + 加入；未細列搜尋列不顯示熱量、確認區塊營養格線與手動編輯等互動。

## 實際做法

1. **搜尋結果列**：僅品名（13px／500）、來源色點（衛福部／USDA／AI／其他）、品牌（11px secondary）；選取時背景 `#E0F5EE`、左側 2px `#1B7A5A`。
2. **確認區**：快速份量按鈕（`overflow-x: auto` + flex）、數字份量 + g/ml 切換（不切換換算數值）、依份量即時計算四項巨量營養素（整數四捨五入）、2×2 可 inline 編輯；次要營養素（纖維、鈉）僅於有資料時顯示一行小字。
3. **AI 估算**：警示框背景 `#FDF0D5`、邊框 `#FAC775`、圓角 12px；維持勾選確認後才可加入。
4. **加入按鈕**：全寬 Primary，文案為「加入{餐次中文名稱}」。
5. **`food_log_items`**：新增 `is_verified`（BOOLEAN，預設 TRUE）；若使用者曾手動編輯任一巨量營養素儲存格，寫入 `false`。
6. **寫入**：`addFoodFromSearchAction` 改為接受前端提交的整數巨量營養素與選用纖維／鈉（依 per 100g 換算）、份量與 `is_verified`。

## 原因

對齊產品規格（先選品項再揭露與確認營養、份量與來源語意更清晰），並讓「手動微調營養數值」在資料層可追溯。
