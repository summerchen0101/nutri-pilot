# 全站卷軸隱藏、血糖開關、運動估計熱量、橘色洞察卡、紀錄 Tab

**日期**：2026-04-21  
**影響規格**：docs/09-ui-design.md、docs/03-features.md（相關頁面行為）  
**類型**：修改

## 原規格

- UI 規範中「AI 建議／洞察」區塊以藍色系底色與強調（`#E6F1FB`、`#378ADD` 等）。  
- 橫向商品／品牌列表未規定是否顯示卷軸；運動紀錄估計熱量為使用者手填為主。

## 實際做法

- **卷軸**：`html`／`body` 與橫向 `overflow-x-auto` 區塊（首頁推薦／人氣、商城精選品牌、商品頁同品牌）套用隱藏卷軸樣式，仍可捲動。  
- **設定**：血糖相關開關調整 thumb 位移與列對齊（`items-start`、文案區 `pr`）。  
- **運動**：新增 `src/lib/activity/kcal-per-minute.ts`（各類型約 kcal／分）；`activity-log-section` 依類型×分鐘自動帶入估計消耗，使用者可修改；送出後仍清空表單。  
- **色彩**：分析頁「AI 週報洞察」區塊、`WeeklyReportShare` 標題、首頁「本週補給推薦」橫幅改為橘色系（淡底 `#FFF4E8`、邊框 `#F0C896`、強調 `#EF9F27`／`#C2410C`）；洞察列表 `info` 類型圓點改橘色。  
- **紀錄頁**：頂部 Tab 改為帶邊框的 segment 容器，各 Tab 含圖示（飲食／運動／標籤）。

## 原因

產品需求：全站不顯示卷軸、修正開關跑版、運動熱量可依表自動試算、週報與補給橫幅改橘色主視覺、紀錄 Tab 更易辨識。

## 後續

可將 `docs/09-ui-design.md` 中「AI 卡藍色」改為橘色敘述，並補充運動估計熱量為介面估算非醫療建議。
