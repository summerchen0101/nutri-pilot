# 飲食紀錄：食物項目與每餐小計顯示巨量／次要營養素

**日期**：2026-04-19  
**影響規格**：docs/06-pages.md（`/log`）、docs/09-ui-design.md  
**類型**：修改

## 原規格

- `docs/06-pages.md`：`/log`「今日記錄列表」為依餐次分組，每筆顯示熱量。
- 實作先前僅列出食物名稱、份量與單筆熱量；每餐卡片小計僅總熱量。

## 實際做法

1. **`food_log_items` 查詢**：於 `app/(main)/log/page.tsx` 一併選取 `fiber_g`、`sodium_mg`，並寫入 `LogItemSnapshot`。
2. **列表每一筆**：固定一行顯示 `kcal`、蛋白質／碳水／脂肪（整數、`·` 分隔）；熱量為 `#1B7A5A`，其餘為 `var(--color-text-secondary)`（含 fallback）；字級 11px。
3. **次要營養素**：若有可呈現之纖維或鈉（非 null 且四捨五入後非 0），顯示「更多 ›／收合 ‹」；展開區以 `max-height` 過渡 150ms、`ease`。纖維／鈉為 null 時顯示「—」，為 0 時顯示 `0g`／`0mg`。`food_log_items` 目前無 `sugar_g` 欄位，展開列**不**顯示糖（僅纖維與鈉）。
4. **每餐紀錄卡小計**：改為該 log 內項目之總熱量＋蛋白質／碳水／脂肪加總，格式與單筆主列一致。

## 原因

- 對齊飲食紀錄頁面與 UI 規格：一眼讀取四大巨量營養素，纖維／鈉可摺疊以保持密度。
- 不重複 migration：糖欄待 schema 具備後再擴充顯示即可。

## 後續

- 若日後新增 `sugar_g`（或類似欄位），可於展開列補上「糖」，並調整「更多」按鈕是否出現的判斷。
- 可視需要將 `docs/06-pages.md` `/log` 表格中「每筆顯示熱量」更新為「巨量營養素＋可摺疊次要營養素」。
